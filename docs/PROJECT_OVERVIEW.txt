================================================================================
  Bradford Council AI Chatbot — Project Overview
================================================================================

PURPOSE
-------
An AI-powered conversational assistant that helps Bradford Council residents
access and navigate local government services. Citizens ask questions in plain
English and receive accurate, structured guidance — including live data lookups,
multi-step dialogue flows, RAG-enhanced answers, and rich HTML response cards.


TECH STACK
----------
  Backend         : Python 3.11 · FastAPI · Uvicorn
  Frontend        : React 18 · Tailwind CSS
  Intent Model    : Token-overlap bigram classifier (~800 labelled examples, 61 intents)
  RAG Pipeline    : Keyword match → embedding re-rank → gpt-4o-mini enhancement
  Embeddings      : OpenAI text-embedding-3-small · FAISS · auto-cached at startup
  Live Data       : Bradford Council website scraping (bin dates, council tax bands)
  External Links  : GOV.UK Blue Badge portal · Turn2Us benefits calculator


SERVICES SUPPORTED
------------------
  1. Bin Collection
     - Next collection dates (live multi-step form scrape of Bradford Council site)
     - Which bin to use for each material (recycling guidance by item)
     - Report missed bins, service disruptions, assisted collections
     - Outside Bradford area guidance

  2. Council Tax
     - Band lookup (live scrape from VOA / GOV.UK)
     - Annual charge by band (2026-27 Bradford figures)
     - Payment methods, direct debit, change payment date / instalments
     - Discounts: single person, student, empty property
     - Council Tax Reduction support, arrears help, appeals, change of address

  3. Benefits Support
     - Housing Benefit eligibility (multi-step clarification flow)
     - Housing Benefit + CTR application guidance
     - Discretionary Housing Payment (DHP)
     - Benefits calculator (links to Turn2Us)
     - Report change of circumstances, evidence submission
     - MyInfo online account access, benefit appeals

  4. Blue Badge
     - Eligibility wizard (4-step yes/no flow: PIP/DLA → walking → hidden disability → child)
     - Renewal reminder flow (parse expiry date, advise urgency, link to GOV.UK)
     - Application guidance, parking rules, cost, replacement, misuse reporting
     - Organisation applications, contact team

  5. Library Services
     - Library finder (by name, area, postcode — sorted by distance)
     - Full library cards: opening hours, facilities, events, contact
     - Membership, catalogue search, eBooks (BorrowBox), book renewals
     - Home Library Service, library events, computers/wifi, meeting rooms

  6. School Admissions
     - School finder (by name or postcode)
     - Primary and secondary admissions guidance
     - Key dates and deadlines, appeals process, in-year transfers
     - Admissions criteria, special educational needs (SEN/EHC), late applications

HOW INTENT CLASSIFICATION WORKS
---------------------------------
  1. User sends a message to POST /chat
  2. ChatEngine normalises text and checks session state:
       - If a pending_action exists (e.g. awaiting postcode), dispatch to that handler
       - If a blue_badge_flow_stage is set, route to BlueBadgeHandler
  3. Otherwise IntentHandler classifies via two mechanisms in order:
       a. Keyword override (intent_keyword_matcher.py) — hard-coded phrase lists
          per service; fast, deterministic, takes priority when matched
       b. Token-overlap bigram classifier — scores user input against ~800 labelled
          training examples; returns top-3 candidates with confidence scores
  4. Confidence routing:
       - < 0.20  → ask user to clarify (too ambiguous)
       - 0.20–0.35  → confirm intent with user before proceeding
       - >= 0.35  → proceed directly
  5. DialogueHandler routes the confirmed intent to:
       - A structured JSON dialogue flow (via dialogue_manager)
       - A live data handler (bin, council tax)
       - A specialist handler (benefits, blue badge, library, school)
       - RAG: keyword FAQ search → embedding re-rank → LLM enhancement


RAG PIPELINE
-------------
  1. User query matched against all FAQ entries for the active service
  2. Keyword scoring filters to top candidates
  3. OpenAI embedding re-ranks to pick the closest semantic match
  4. ResponseEnhancer calls gpt-4o-mini with a service-specific prompt template
     (from llm/prompts.py) to rephrase the raw FAQ answer into a natural reply
  5. Embedding caches are built at startup (api.py background thread) so first
     queries do not pay the embedding generation cost


CONFIDENCE THRESHOLDS
---------------------
  LOW_CONFIDENCE_THRESHOLD  = 0.20   below this → ask user to clarify
  MID_CONFIDENCE_THRESHOLD  = 0.35   above this → proceed without confirmation
  (between 0.20 and 0.35)            → confirm intent with user before proceeding


EVALUATION RESULTS
------------------
  Intent classifier (held-out eval sets, 328 examples, 61 intents):
    Top-1 Accuracy : 95.1%
    Macro F1       : 0.94
    Weighted F1    : 0.95

  End-to-end chatbot evaluation (22 manual test scenarios):
    Overall pass rate      : 77% (17/22)
    Intent classification  : 86% (19/22)
    Service routing        : 91% (20/22)
    RAG answer accuracy    : 89% (8/9)
    Multi-turn completion  : 71% (5/7)

  See evaluation/classification_report.txt for full per-intent breakdown.
  Run `python generate_eval_charts.py` for poster-ready PNG charts.


ADDING A NEW SERVICE
---------------------
  1. Create datasets/<service_name>/<service_name>_intents.json
  2. Create datasets/<service_name>/<service_name>_faq.json
  3. Optionally create datasets/<service_name>/<service_name>_dialogue.json
  4. Add service to SERVICE_OPTIONS in backend/chat_config.py
  5. Add intent list to INTENT_GROUPS in backend/chat_config.py
  6. Add intent labels to INTENT_LABELS in backend/chat_config.py
  7. Add keyword phrases to backend/handlers/intent_keyword_matcher.py
  8. Add LLM prompt template to backend/llm/prompts.py
  9. Run: python scripts/train_model.py
